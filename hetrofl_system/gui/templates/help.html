Getting Started

Welcome to HETROFL!

This guide will help you get started with the Heterogeneous Federated Learning system for IoT network security.

System Requirements
Minimum Requirements
  • Python 3.8+
  • 4GB RAM
  • 2GB Storage
  • Modern web browser
Recommended
  • Python 3.10+
  • 8GB+ RAM
  • 10GB+ Storage
  • GPU support (optional)
Quick Setup Steps
1
Install Dependencies

Install required Python packages using pip:

pip install -r requirements.txt
2
Configure System

Edit the configuration file to match your environment:

python configure.py
3
Start the System

Launch the HETROFL system:

python run_hetrofl.py
4
Access Dashboard

Open your web browser and navigate to:

http://localhost:5000
First Time Setup Complete!

You should now see the HETROFL dashboard. Continue with the guided tour to learn about the system features.

Model Management

HETROFL supports multiple types of machine learning models working together in a federated learning setup.

Supported Models
XGBoost Classifier

Gradient boosting framework optimized for speed and performance. Excellent for structured data and feature importance analysis.

Random Forest

Ensemble method that combines multiple decision trees. Robust against overfitting and handles missing values well.

CatBoost

Advanced gradient boosting with automatic categorical feature handling. Reduces overfitting and provides fast training.

Global MLP

Multi-layer perceptron neural network that aggregates knowledge from all local models through federated learning.

Model Operations
Loading Models

Models are automatically loaded from their respective directories. Ensure model files are in the correct format and location.

Model Updates

Models are updated during federated training rounds. Progress can be monitored in real-time through the dashboard.

Performance Monitoring

Track model performance metrics including accuracy, F1-score, precision, and recall for each model type.

Training Process

The federated training process involves multiple stages and can be controlled through the training interface.

1

→
2

Train individual models (XGBoost, Random Forest, CatBoost)

→
3
Knowledge Distillation

Transfer knowledge from local models to global model

→
4
Global Aggregation

Update global model with aggregated knowledge

Training Parameters
Sample Size

Number of data samples to use for training. Larger samples provide better accuracy but require more time.

Default: 50,000 Range: 1,000 - 500,000
Training Rounds

Number of federated learning rounds to perform. More rounds generally improve performance.

Default: 50 Range: 1 - 200
Learning Rate

Controls how quickly the model adapts to new information. Lower values provide more stable training.

Default: 0.01 Range: 0.001 - 0.1
Starting Training
  1. Navigate to the Training page
  2. Configure training parameters
  3. Click "Start Training" button
  4. Monitor progress in real-time
  5. View results in the Metrics section
Important Notes
  • Training can take several minutes to hours depending on data size
  • Ensure sufficient system resources are available
  • Training can be stopped at any time without losing progress
  • Results are automatically saved and can be exported

Metrics & Analytics

The metrics section provides comprehensive analytics and performance insights for your federated learning system.

Available Metrics
Accuracy

Percentage of correct predictions. Higher values indicate better model performance.

F1-Score

Harmonic mean of precision and recall. Provides balanced view of model performance.

Precision

Ratio of true positive predictions to total positive predictions. Measures prediction quality.

Recall

Ratio of true positive predictions to actual positive cases. Measures detection capability.

API Documentation

HETROFL provides a comprehensive REST API for integration with external systems and programmatic access.

Base URL
http://localhost:5000/api
Authentication

Currently, the API uses basic authentication. Include your credentials in the request headers.

Main Endpoints
System Status
GET /api/status

Get current system status and health information

Models
GET /api/models/info

Get information about all loaded models

GET /api/models/performance

Get detailed performance data for all models

Training
POST /api/training/start

Start federated training with specified parameters

POST /api/training/stop

Stop current training process

Metrics
GET /api/metrics/latest

Get latest performance metrics for all models

GET /api/metrics/history

Get historical metrics data

GET /api/metrics/improvements

Get improvement percentages for all models

Example Request
curl -X GET "http://localhost:5000/api/status" \
     -H "Content-Type: application/json"

Troubleshooting

Common issues and their solutions to help you resolve problems quickly.

Common Issues
System Won't Start

Symptoms: Error messages during startup, system fails to initialize

Solutions:

  • Check Python version (3.8+ required)
  • Verify all dependencies are installed
  • Ensure sufficient disk space and memory
  • Check file permissions in the project directory
python --version && pip list
Models Not Loading

Symptoms: Model status shows "not loaded", missing model information

Solutions:

  • Verify model files exist in correct directories
  • Check file formats and compatibility
  • Ensure proper file permissions
  • Try rebuilding models using the rebuild function
Training Takes Too Long

Symptoms: Training process is very slow or appears stuck

Solutions:

  • Reduce sample size for faster training
  • Decrease number of training rounds
  • Check system resources (CPU, memory usage)
  • Close other resource-intensive applications
No Metrics Data

Symptoms: Charts show no data, metrics appear as zeros

Solutions:

  • Complete at least one training round
  • Check if training completed successfully
  • Refresh the metrics page
  • Verify data files are being created

Frequently Asked Questions

What is heterogeneous federated learning?

Heterogeneous federated learning allows different types of machine learning models to collaborate and share knowledge without sharing raw data. In HETROFL, models like XGBoost, Random Forest, and CatBoost work together to improve overall performance while maintaining data privacy.

How long does training typically take?

Training time depends on several factors including data size, number of rounds, and system specifications. Typically, training with 50,000 samples for 50 rounds takes 15-30 minutes on a modern system. You can reduce sample size or rounds for faster training.

Can I use my own dataset?

Yes, you can use your own IoT network security dataset. Ensure it follows the expected format with proper feature columns and target labels. The system supports CSV format with standard network security features.

What happens if training is interrupted?

The system automatically saves progress after each training round. If training is interrupted, you can resume from the last completed round without losing progress. All metrics and model states are preserved.

How do I interpret the performance metrics?

Key metrics include: Accuracy (overall correctness), F1-Score (balanced performance), Precision (prediction quality), and Recall (detection capability). Higher values generally indicate better performance. The metrics dashboard provides detailed explanations and comparisons.

Is the system suitable for production use?

HETROFL is designed as a research and development platform. While it includes production-ready features like API endpoints and real-time monitoring, additional security and scalability considerations may be needed for production deployment.

Can I add new model types?

Yes, the system is designed to be extensible. You can add new model types by implementing the model adapter interface and updating the configuration. Detailed instructions are available in the developer documentation.
